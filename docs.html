Documentation

Blabble was designed to be cross-platform and cross-browser compatible. It is currently used in a live environment on Windows 7 computers running as part of a Google Chrome extension. While FireBreath produces a valid NPAPI plugin and ActiveX control, Blabble has not been tested under Internet Explorer. Primary testing and development has been done under Google Chrome on Windows 7.

 

Compiling

Windows

Visual Studio is used to compile on Windows. Currently, Visual Studio 2012 Professional was used to compile the binaries. Other versions should work but have not been tested.

PJSIP

A compiled version of PJSIP is included in the git repository under the "external" folder. If you wish to compile and use your own version of PJSIP (to enable TLS or customize other features, see here). The compiled version can then be placed into the "external" folder. If you changed the file name be sure to update the library reference in CMakeLists.txt. If you are using a newer version of PJSIP, you will need to update the included header files in "external/pjsip". Currently, Blabble is compiled against SVN revision 4339 of PJSIP. 2.0.1 is not used because of some bugs related to ICE.

FireBreath

FireBreath uses CMake and Visual Studio to compile the plugin. See here for their instructions. Binaries are currently compiled with Visual Studio 2012 Professional. To compile, first download FireBreath (currently binaryies are compiled against version 1.7). Create a folder named projects and checkout the Blabble repository into the folder. Next, run prep2012.cmd to create the project files in the build directory. Open the resulting FireBreath.sln solution in Visual Studio and build. The plugin will be located in ./Build/bin/Blabble/[BuildType]/ where BuildType will be something like Debug or Release depending on what you picked.

Linux

Blabble has been tested to compile under Fedora 17, Fedora 18, and Ubuntu 12.10 though any semi-recent distribution should work. The basic requirements are g++ and cmake. Refer to the FireBreath and PJSIP documentation for their requirements. Blabble does not require anything further than what FireBreath and PJSIP require. A static library is provided that should work for most distributions.

Mac OS X

Blabble has been tested to compile under Mountain Lion though previous versions should work too. Xcode, CMake, and pkg-config are required. Refer to the FireBreath and PJSIP documentation for their requirements. Once PJSIP is built, CMake can be used to generate the XCode project for Blabble. A binary is provided for Mac OS X 10.7 or later.

 

Distributing

There's many ways you can distribute the plugin. It can be embedded in a Google Chrome plugin, Firefox plugin, or installed and registered as a NPAPI plugin. An example Google Chrome plugin is available in the downloads section.
